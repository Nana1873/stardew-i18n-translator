# Stardew i18n Translator v1.4.0

This release expands the language model for translators who work beyond
Stardew Valley's built-in locales, and tightens glossary hints so official game
terms stay useful without adding noisy matches.

## Highlights

- Target languages are now data-driven, with Thai (`th`) available as the first
  custom/unsupported language target. The app still edits and exports only SMAPI
  `i18n/<lang>.json` files.
- Game-unsupported languages can translate, export, use external LLM batches,
  and use local AI even when Stardew has no official locale data for them.
- Glossary caches are now per language under
  `data/glossary/glossary-<lang>.json`, so switching languages cannot reuse
  another language's official terms.
- Glossary entries now use a typed format with high-confidence term categories
  such as items, craftables, weapons, tools, clothing, NPCs, locations, and
  seasons.
- Older glossary cache layouts are ignored or migrated safely, and the UI
  recommends rebuilding when it finds a cache from an earlier format.
- Installed community Content Patcher language packs are skipped during normal
  mod scanning and can be used read-only as glossary sources for unsupported
  languages when they register the selected language.

## Upgrade Notes

- Replace the previous portable application files with the v1.4.0 package.
- Keep the existing `data/` or `Data/` directory to preserve settings,
  glossary data, and translation progress. Windows treats the folder name as
  the same path.
- For built-in Stardew languages, rebuild the glossary to get the new typed,
  high-confidence cache format.
- For custom/unsupported languages such as Thai, Stardew may not provide
  official glossary extraction. If no installed community language pack
  registers that language, glossary hints remain unavailable while translation,
  editing, export, batch, and local-AI workflows continue normally.
- No user data, settings, glossary cache, or translation working state is
  included in the portable release ZIP.

The application remains a portable, local-first Windows tool without telemetry,
cloud credentials, Nexus API access, or automatic uploads.
