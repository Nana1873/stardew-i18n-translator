# Stardew i18n Translator v1.2.3

This patch release improves configuration durability, restores cross-mod
search, and removes two workflow inconsistencies found during the v1.2.x
review.

## User-Facing Fixes

- Settings are written atomically. Before an overwrite, the last valid
  `Data/settings.json` is retained as `Data/settings.json.bak`.
- If the main settings file is damaged, the app recovers from the valid backup.
  When both files are unusable, it reports the problem instead of silently
  resetting paths, language, and local-AI settings.
- Adding, removing, or restyling paired single-quote delimiters now produces a
  soft review warning. Quote punctuation no longer blocks export or causes a
  local-AI retry.
- Bulk **Mark as translated** keeps rows with an empty target untranslated, so
  status filters and progress counts remain consistent.
- The toolbar search can once again search keys, source text, and translations
  across every scanned mod. Results show their mod and file and open directly
  into the matching per-mod view.

## Project Maintenance

- Complete remote CI now runs once on the exact `main` commit instead of being
  duplicated for both pull requests and merges.
- Obsolete CI and pull-request policy runs are cancelled automatically.
- Dependency audits run weekly or on manual request.
- Release drafts upload the exact locally built and smoke-tested portable ZIP,
  avoiding a duplicate Windows Actions build.

## Upgrade Notes

- Replace the previous portable application files with the v1.2.3 package.
- Keep the existing `Data/` directory to preserve settings, glossary data, and
  translation progress.
- No data migration is required. `settings.json.bak` is created automatically
  after a later successful settings overwrite.

The application remains a portable, local-first Windows tool without telemetry,
cloud credentials, or Nexus API access.
