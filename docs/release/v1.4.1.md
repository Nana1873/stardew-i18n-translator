# Stardew i18n Translator v1.4.1

This patch release extends the custom-language workflow introduced in v1.4.0 and
makes glossary building work directly from local Stardew and community language
pack XNB string dictionaries.

## Highlights

- Added curated custom-language targets for Vietnamese (`vi`), Indonesian
  (`id`), Ukrainian (`uk`), Polish (`pl`), Finnish (`fi`), Dutch (`nl`), and
  Czech (`cs`).
- Verified the new custom targets through the same SMAPI `i18n/<code>.json`
  workflow as built-in languages: scan/import, saved edits, export, external LLM
  batch import/export, and local-AI prompts.
- Glossary building now reads official Stardew `Content/Strings/*.xnb`
  dictionaries directly for built-in game languages.
- The previous `Content (unpacked)/Strings/*.json` glossary source remains as a
  compatibility fallback.
- Installed community language packs can now supply glossary terms from either
  JSON `Strings/*.json` files or direct `Strings/*_<lang>.xnb` dictionaries.
- Setup can auto-build a glossary when a readable local game or pack source is
  available; missing or failed glossary sources still never block setup,
  translation, or export.
- Release publication now includes a Nexus Mods upload workflow for the already
  verified portable ZIP after a normal GitHub release is published.

## Upgrade Notes

- Replace the previous portable application files with the v1.4.1 package.
- Keep the existing `data/` directory to preserve settings, glossary caches, and
  translation progress.
- Rebuild the glossary from Settings after updating Stardew Valley or an
  installed community language pack.
- Custom/unsupported languages still require a Stardew custom-language mod to be
  playable in-game. The translator can create and edit the corresponding
  `i18n/<code>.json` files even when no glossary source is installed.
- No user data, settings, glossary cache, or translation working state is
  included in the portable release ZIP.

The application remains a portable, local-first Windows tool without telemetry,
cloud credentials, Nexus API access from the app, or automatic in-app downloads.
