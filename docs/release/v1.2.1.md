# Stardew i18n Translator v1.2.1

A small correctness patch. No new features, no behavior changes to the normal
translate/export loop.

## Fixes

- **Clearing every translation now removes the stale file.** When you clear all
  of a mod's translations and export, the old `i18n/<lang>.json` is removed
  (after a `.bak` backup) instead of being left on disk. Previously the file was
  simply not rewritten, so SMAPI kept loading the outdated translation. Now the
  mod cleanly falls back to English.
- **Duplicate mod UniqueIDs are now reported.** If two scanned mods declare the
  same `UniqueID`, the scan shows a warning instead of silently merging their
  saved progress into one state file (which could overwrite your work). SMAPI
  itself refuses to load duplicate UniqueIDs, so this surfaces a broken or
  duplicated install you can fix.
- **Pre-existing translations can now go outdated.** A community translation you
  downloaded (a `<lang>.json` you never edited in the app) had no record of the
  English source it was translated against, so it could never be flagged
  outdated. Now, the first time you open the mod, each such string is given a
  baseline of the current source text. If the mod later updates and that source
  changes, the string is correctly shown as **outdated** for review. (Strings
  you edit in the app already worked this way.)

## Notes

- Both changes preserve the app's safety guarantees: removals are backed up to a
  `.bak` sibling first, and the duplicate-UniqueID case is only ever a warning —
  nothing is deleted or merged automatically.
- Everything remains local and offline. No telemetry, no network.

## Installation

Download and extract the portable Windows ZIP to a writable folder, then run
`stardew-i18n-translator.exe`. Keep the adjacent `Data` folder with the
executable when moving the application. The executable is unsigned, so Windows
SmartScreen may show an unknown-publisher warning.
