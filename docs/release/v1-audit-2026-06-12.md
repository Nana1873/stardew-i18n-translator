# V1 Release Audit — 2026-06-12

## Result

The v1 application scope is functionally complete. The core workflow and both
AI workflows are implemented, frontend and Rust test suites pass, and the
Windows Tauri release executable builds successfully.

## Checks

- Prettier, TypeScript, 124 Vitest tests, and Vite production build.
- Rust format, Clippy with warnings denied, and the complete locked test suite.
- Tauri release build without installer bundling.
- Release executable startup smoke test (12,773,888-byte Windows EXE).
- Milestone, README, SPEC, status-model, and deferred-feature reconciliation.

## Findings Resolved

- Updated stale five-status / `not-translatable` documentation to the shipped
  four-status + Keep original model.
- Removed already shipped local-AI and `newline-mismatch` work from the v1.1
  backlog.
- Marked the v1 and external-batch feature checklists complete.
- Fixed two Clippy warnings in Rust tests.
- Added Rustfmt and warnings-as-errors Clippy checks to GitHub CI.
- Excluded local `.claude/` and `.preview/` workspaces from Prettier checks.
- Polished the first-run setup with persistent progress, clearer folder and
  language states, responsive layout, and consistent primary actions.
- Added an in-app glossary how-to covering StardewXnbHack preparation, local
  extraction, translation hints, privacy, and the optional later workflow.

## Release Packaging Decision

- The first distributable is version `1.0.0`.
- The initial package is a portable Windows ZIP containing the executable and
  its adjacent `Data/` folder.
- Tag builds create draft GitHub releases and remain manually publishable.
- V1 is unsigned; the expected SmartScreen warning is documented.

## Remaining Release Gate

  folder and one extracted-ZIP persistence smoke test before creating the public
  `v1.0.0` tag.
