# V1.1.0 Release Preparation Plan

## Scope

Prepare the completed v1.1 work for a portable Windows release:

- Synchronize application version metadata to `1.1.0`.
- Publish concise release notes for users upgrading from `1.0.1`.
- Document the language-state migration and unsigned portable package.
- Run the complete frontend, Rust, production-build, and packaging checks.
- Merge a release-preparation PR before creating the `v1.1.0` tag.

## Release Contents

- Configurable keyboard shortcuts with conflict detection and reset.
- Drag-and-drop import for external LLM batch results.
- Inline editing for short, single-line translations.
- Blocking protected-token count validation before any mod export writes.
- Supported-language verification for all 11 target languages.
- Portuguese `pt-BR.json` import fallback with canonical `pt.json` export.
- Per-language saved working state with one-time legacy-state migration.
- Updated README, screenshots, license metadata, and Settings About page.

## Non-Goals

- Publishing the draft GitHub release without a final human review.
- Code signing.
- Nexus integration or cloud-AI credentials.
