# V1.1 Candidate Backlog

This list was reconciled during the v1 release audit on 2026-06-12. It removes
items that older drafts still called "v1.1" but which already shipped in v1,
including local AI, batch local-AI translation, full protected-token handling,
`newline-mismatch`, the dashboard, and the settings redesign.

## Strong Candidates

| Candidate                       | Why it is still useful                                                    | Main risk                                                         |
| ------------------------------- | ------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| Inline table editing            | Faster corrections for short labels without opening the full editor       | Must preserve validation, keyboard navigation, and virtualization |
| Supported-language verification | Confirms every advertised target language works through the full workflow | Avoid confusing technical compatibility with translation quality  |

## Nexus Track

M5 remains a separate, larger decision:

- Nexus API key storage and validation.
- Mod metadata enrichment.
- Heuristic translation discovery with user confirmation.
- Premium download plus non-Premium browser fallback.

This should not be mixed into a small v1.1 convenience release unless the
product owner deliberately prioritizes the full M5 workflow.

## Planned Order

1. Inline table editing.

`glossary-deviation` is deliberately removed. Glossary matches are translation
hints, not grammatical rules; inflection and context make exact substitutions
too noisy for validation.

## V1.1 Release Gate: Supported Languages

Before v1.1 is released, all advertised target languages must pass the
compatibility matrix in
[v1.1-language-compatibility.md](v1.1-language-compatibility.md).

This is a release-quality task rather than a new screen or a claim that the app
can judge translation quality. It verifies the technical workflow for `de`,
`es`, `fr`, `hu`, `it`, `ja`, `ko`, `pt`, `ru`, `tr`, and `zh`, including
non-Latin text and Portuguese filename fallback behavior.

## Completed During Release Hardening

- Configurable shortcuts with conflict detection and reset to defaults.
- Drag-and-drop LLM result import with clear rejection feedback.
- Exact protected-token count validation in both directions.
- Complete affected-mod export blocking before writes or backups.
- Non-blocking `identical-to-source` and `escape-suspicious` warnings.
