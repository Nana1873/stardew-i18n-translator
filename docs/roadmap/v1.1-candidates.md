# V1.1 Candidate Backlog

This list was reconciled during the v1 release audit on 2026-06-12. It removes
items that older drafts still called "v1.1" but which already shipped in v1,
including local AI, batch local-AI translation, full protected-token handling,
`newline-mismatch`, the dashboard, and the settings redesign.

## Strong Candidates

| Candidate                       | Why it is still useful                                                                                   | Main risk                                                         |
| ------------------------------- | -------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| Configurable shortcuts          | Explicitly requested and already reserved as a Settings page in SPEC §7.7                                | Conflict detection and platform-specific key combinations         |
| Focused quality validation      | Add only high-signal checks such as `glossary-deviation`, `identical-to-source`, and suspicious escaping | Too many warnings would make review noisy                         |
| Drag-and-drop LLM result import | Removes the file-picker step from the established batch workflow                                         | Tauri drag/drop path handling and clear rejection feedback        |
| Inline table editing            | Faster corrections for short labels without opening the full editor                                      | Must preserve validation, keyboard navigation, and virtualization |

## Nexus Track

M5 remains a separate, larger decision:

- Nexus API key storage and validation.
- Mod metadata enrichment.
- Heuristic translation discovery with user confirmation.
- Premium download plus non-Premium browser fallback.

This should not be mixed into a small v1.1 convenience release unless the
product owner deliberately prioritizes the full M5 workflow.

## Optional / Lower Priority

- Cloud-AI credentials for OpenAI-compatible remote endpoints.
- Theme toggle.
- `extra-key` reporting.
- Finalize-and-propagate identical strings across mods (currently v2).

## Already Shipped Or Superseded

- Local AI translate button and batch translation.
- `newline-mismatch`.
- Full bracket/dialogue/token protection; a separate
  `bracket-token-missing` rule would duplicate `token-missing`.
- Dashboard and settings left navigation.
- Five-status model; v1.5 intentionally has four statuses.
