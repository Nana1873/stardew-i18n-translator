# v1.2.0 Plan

## Goal

Improve diagnostics and privacy control without expanding the portable
translator into a mod manager or cloud service.

## Scope

### Extra-Key Diagnostics

- Detect keys present in an existing target file but absent from
  `default.json`.
- Report them under a neutral **Optional cleanup** section during scan/import.
- Never import or export those keys automatically.
- Explain that SMAPI ignores them and show the affected file and key.
- Do not block editing or export.

### Local Logging Control

- Add an enable/disable switch under **Settings > About**.
- Keep logging enabled by default.
- Store the preference locally in the portable `Data/settings.json`.
- Explain that disabling logs reduces diagnostic information available for bug
  reports.
- Never add telemetry, remote logging, crash uploads, or network reporting.

## Explicit Non-Goals

- No theme toggle or additional themes. The warm dark theme is the fixed
  product design.
- No custom or non-standard language codes. The app supports only Stardew
  Valley's verified language set.
- No Nexus API integration, discovery, or downloads. M5 is the v1.3 project.
- No cloud-AI credentials or remote provider integration.
- No translation-quality, grammar, or glossary-compliance scoring.
- No mod-manager, publishing, Git, or cloud-sync features.

## Tests

- Extra target keys are reported with file and key details but are neither
  imported nor exported.
- Extra-key diagnostics do not block export.
- Logging preference survives restart and prevents new application log entries
  while disabled.
- Existing `de`, `ja`, and `pt` smoke paths remain unchanged.

## Release Gate

- Full TypeScript and Rust test suites pass.
- Production portable package contains no user data.
- Manual smoke test covers scan, extra-key reporting, export, and logging
  control.
- README and changelog explain extra-key reporting and logging control.

## Deferred Beyond v1.2

- Nexus integration: v1.3 / M5.
- Optional cloud-AI credentials: unscheduled.
- Cross-mod translation memory and finalize-and-propagate: v2.
- Content Patcher and arbitrary data-file translation: v2+.
- In-app XNB decoding: future version.
